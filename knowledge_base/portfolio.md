# Projects
## More3zdenAI – Conversational Portfolio Assistant
Tech: Python, FastAPI, Next.js, PostgreSQL, FAISS, Ollama, Redis, Docker. This is the assistant you're talking to right now. Instead of clicking through a static portfolio, visitors can just ask questions in plain language and get grounded answers. Under the hood it's a Retrieval-Augmented Generation (RAG) system: I index a structured knowledge base into a FAISS vector store, retrieve the most relevant context, and have a local LLM generate answers strictly from that context — which keeps responses accurate and on-topic. I added Redis caching and prompt engineering to keep latency low, and the whole stack is containerized with CI/CD so it stays easy to deploy and maintain. It basically turns my portfolio into something you can have a conversation with.

## Droove – Intelligent Ride-Sharing Platform
Tech: Spring Boot, NestJS, React, Microservices, Docker, AWS, PostgreSQL, MongoDB. Droove is a ride-sharing platform I designed around microservices, with modularity and fault tolerance as first-class goals. It does real-time driver tracking using WebSockets and MongoDB geospatial queries, and the architecture is intentionally AI-ready so route optimization and demand prediction can be plugged in later. I deployed the containerized services on AWS (ECS and RDS) for high availability, and automated testing and deployment with GitLab CI, Maven, and SonarQube.

## Multi-Cloud Cost Intelligence Platform
Tech: Spring Boot, React, Docker, AWS, GCP, Azure, PostgreSQL. This one pulls billing data from AWS, GCP, and Azure into a single place so you can actually understand where the money is going. I built analytics services to surface spending trends, usage anomalies, and optimization opportunities, plus a real-time dashboard for cost visibility, forecasting, and budget alerts. I also laid the groundwork for ML-based cost prediction and anomaly detection, and deployed the microservices on AWS with CI/CD.
